使用github+hexo+next搭建博客

Posted on 2016-11-13

作为一个医学生，感觉自己不好好用farbox这种小白也能上手的托管平台而是作死去鼓捣hexo真是不作死就不会死，说好的搭建起来非常简单呢？简直步步惊心web版好吗？要不是farbox的流量费用太高（其实很便宜了）+对于高层级的list编译非常不好（和typora差了一个空格），我是不会想到要另寻出路的。但没办法，什么都可以放弃，就是markdown不行，就是typora不行（实在是太好用了）。所以看了李笑来老师写的利用hexo和github建站就动心了，毕竟这次不是鼓捣数据库blabla的了，又不是用wordpress（而且那么难用），github完全能hold得住这种静态网站，于是展开了自己的作死之旅，特此写一篇总结以示后人。

必须环境的搭建

git的安装

光git的安装就已经是一步深坑了，哪怕是全局ss（科学上网）也没办法成功下下来git的最新版，无奈求助于国内镜像站点：git镜像，虽然因为不相信自己打开了以后是git bash没有错，又咬着牙搭着ss下了一遍官网的git，最终才确定无误安装成功。安装过程倒是比较简单，只要一路next下去就可以了，不需要设置环境变量。

关于git的语言入门，推荐git - 简明指南或pro git简体中文版，其实暂时也用不到这种教程，只是怕跟我一样的小白被命令行吓哭。

  • git - 简明指南
  • pro git简体中文版

node.js的安装

下载地址：node.js官网，不要去百度搜到的node.js中文网上下载，下的文件对不上的。

一路next安装过来即可。设置环境变量似乎也不需要（如果后面的步骤出现问题，可以检查一下有没有添加环境变量）。

确定git和node.js的安装成功

在git bash中分别输入

node.js中输入

node version

分别显示该版本版本号即说明安装成功。可是安装成功不代表你能一会成功从git上下载hexo啊呵呵呵呵呵呵呵呵呵呵呵。

github账号的申请与部署

  • 像申请一般账号一样申请一个github的账号
  • 利用自己的用户名申请一个repositoriy yourname.github.io
    • 请注意，一定是自己的id，不能用别的词来申请。
  • ssh密钥的布置，不设置这个东西等着被烦死吧。

Hexo的安装与部署

  • 如果没有响应，可以选择更换镜像来安装
  • 设置Hexo的个人信息
  • 布置
  • 初步观赏自己的本地网站

更换Hexo模板：Next

  • 安装next
    • 见next官网
  • 显示白屏？更改网站配置文件
  • 按照Hexo手册自定义网站内容
    • 依然大多是next官网就能解决的小问题
  • 关于algolia

最后，一点小小的总结

  • 对于外行人，就应该选择更热门、相对简单的平台，哪怕出现bug也有的可查
  • 排查bug的时候，别用百度用google
  • 有的时候shadowsocks这样科学上网的工具是很必要的，有的时候有ss全局也没用，必须靠镜像
  • 一边用一边学（撸起袖子去学javascript和git啦
  • 出现bug的时候只要想办法，只要钻得起牛角尖，真的总能解决，磕磕绊绊的，还不是建成了这个网站？
